How Java Works?

Review
  1. Compilation Process:
    • Java source code (.java files) is compiled using the javac compiler.
    • The compiler translates the Java source code into bytecode (.class files).
  1. Execution Process:
    • Bytecode is executed by the Java Virtual Machine (JVM).
    • JVM is responsible for loading, verifying, and executing bytecode instructions.
    • JVM looks for a specific entry point in the bytecode, which is denoted by the public static void main method signature.
    • Execution of the program starts from the main method.
  1. Java Runtime Environment (JRE):
    • JVM runs inside the Java Runtime Environment (JRE), which provides the necessary runtime environment for executing Java programs.
    • JRE includes additional Java libraries and resources required for program execution.
  1. Key Components:
    • javac: Java compiler that translates Java source code into bytecode.
    • JVM: Java Virtual Machine responsible for executing bytecode.
    • main method: Entry point of a Java program where execution begins.
    • JRE: Java Runtime Environment that provides the necessary runtime environment for Java programs.
  1. Platform Independence:
    • Java's platform independence allows bytecode to run on any device or platform with a compatible JVM installed.
    • This enables Java programs to be portable across different operating systems and hardware architectures.
  1. Additional Notes:
    • During execution, JVM manages memory, handles exceptions, and performs other runtime tasks to ensure correct program behavior.
    • Java's automatic memory management through garbage collection helps manage memory allocation and deallocation.
    • Security features in Java, such as access controls and sandboxing, protect against common vulnerabilities.

Overall, Java's compilation to bytecode and execution by the JVM in the JRE environment enable platform-independent and secure execution of Java programs.
