Sealed classes in Java

Review

Sealed classes, introduced in Java 17, are a new feature that allows developers to restrict the inheritance hierarchy of a class hierarchy. By marking a class as sealed, you define a limited set of subclasses that can directly extend or implement it. This can help improve the maintainability, readability, and security of your code by explicitly specifying which classes are allowed to extend the sealed class.

Here's an overview of sealed classes in Java:

Syntax:

To declare a sealed class, you use the sealed modifier followed by the permitted subclasses or interfaces within curly braces {}. Each permitted subclass or interface is separated by commas.

public sealed class SealedClass permits Subclass1, Subclass2, ... {
    // Class definition
}

Permitted Subclasses:

  • Each subclass listed after the permits keyword is allowed to directly extend the sealed class.
  • Subclasses must be either final or sealed themselves, or they must be listed as permitted subclasses in their own permits clause.

Example:

public sealed class Shape permits Circle, Rectangle, Triangle {
    // Class definition
}

public final class Circle extends Shape {
    // Class definition
}

public sealed class Rectangle extends Shape permits Square {
    // Class definition
}

public final class Triangle extends Shape {
    // Class definition
}

public final class Square extends Rectangle {
    // Class definition
}

In this example, the Shape class is declared as sealed and permits three subclasses: Circle, Rectangle, and Triangle. The Rectangle class, in turn, permits the Square subclass.

Benefits of Sealed Classes:

  1. Explicit Hierarchy: Sealed classes provide an explicit and controlled hierarchy, making it clear which classes are allowed to extend the sealed class.
  1. Enhanced Readability: By limiting the set of permitted subclasses, sealed classes can improve the readability and understanding of the class hierarchy.
  1. Improved Security: Sealed classes can help enforce design constraints and prevent unintended subclassing, thereby enhancing security.

Considerations:

  • Sealed classes must be declared in non-final classes and must be in the same package as their permitted subclasses.
  • The permitted subclasses can be either in the same source file or in a separate file.

Use Cases:

  • Sealed classes are useful when you want to define a closed set of subclasses for a base class, such as shapes, colors, or account types.
  • They can also be used in API design to enforce compatibility and prevent unintended subclassing by clients.

In summary, sealed classes provide a mechanism for restricting the inheritance hierarchy in Java, improving the clarity, maintainability, and security of your code. They are a valuable addition to the language for designing robust and maintainable class hierarchies.
