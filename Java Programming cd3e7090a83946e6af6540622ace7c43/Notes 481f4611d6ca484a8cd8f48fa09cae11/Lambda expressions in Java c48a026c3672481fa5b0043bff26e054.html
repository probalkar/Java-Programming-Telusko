Lambda expressions in Java

Review

Lambda expressions, introduced in Java 8, are a powerful feature that allows you to treat functionality as a method argument or create instances of functional interfaces concisely and with less boilerplate code. Lambda expressions are used extensively in functional programming and are often associated with streams, collections, and concurrency APIs. Here's an explanation of lambda expressions in Java:

Syntax:

  • Lambda expressions consist of three parts: a parameter list, an arrow (->), and a body.
  • The parameter list represents the input parameters (if any) required by the functional interface's abstract method.
  • The arrow (->) separates the parameter list from the body of the lambda expression.
  • The body represents the implementation of the abstract method of the functional interface.

Syntax of lambda expression:

(parameters) -> expression

or

(parameters) -> { statements; }

Example:

// Lambda expression to square a number
Function<Integer, Integer> square = (x) -> x * x;
int result = square.apply(5); // result = 25

Key Points:

  1. Functional Interfaces: Lambda expressions are closely associated with functional interfaces, which are interfaces with a single abstract method. Lambda expressions provide a way to implement the abstract method of a functional interface concisely.
  1. Concise Syntax: Lambda expressions provide a more concise syntax compared to anonymous inner classes, especially for small, single-method implementations.
  1. Type Inference: In many cases, the type of parameters in lambda expressions can be inferred by the compiler based on the context in which the lambda expression is used. However, explicit type declarations can also be used if needed.
  1. Access to Variables: Lambda expressions can access variables from the surrounding scope, including local variables and instance variables. However, these variables must be effectively final or effectively final (i.e., not modified after initialization).
  1. Functional Programming: Lambda expressions promote functional programming paradigms in Java, allowing for a more declarative and expressive style of coding. They enable operations such as filtering, mapping, and reducing collections using functional interfaces and streams.

Example of Lambda Expression with Collections:

List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5);

// Using lambda expression with forEach method
numbers.forEach((num) -> System.out.println(num));

In the example above, the lambda expression (num) -> System.out.println(num) is passed as an argument to the forEach method of the List interface. It represents a function that takes an integer (num) as input and prints it to the console.

Lambda expressions are a significant enhancement to the Java language, enabling more concise and expressive code, particularly in functional programming scenarios. They play a central role in the development of modern Java applications, facilitating a shift towards more functional and declarative programming styles.

// lambda expressions can only be used with functional interfaces
@FunctionalInterface
interface A{    
    // void show(int i, int j);
    void show(int i);
}

public class Lambda {
    public static void main(String[] args) {
        // A obj = (i,j) -> System.out.println("in show " + (i+j));
        // no need to mention variable datatype when passing arguments
        // also no need of {} when writing only one statement

        // obj.show(5,8);

        A obj = i -> System.out.println("in show " + i);
        // no need to put variable inside () when passing only one argument
        obj.show(5);
    }
}
