Record classes in Java

Created@March 24, 2024 10:44 PM

Review

Record classes, introduced in Java 14 as a preview feature and made stable in Java 16, are a new type of class primarily designed for simple data carrier purposes. They are concise, immutable, transparent, and offer built-in support for features like equals(), hashCode(), and toString(). Records provide a convenient way to model and work with data in Java, reducing boilerplate code and improving readability.

Here are the key characteristics and features of record classes in Java:

Syntax:

To declare a record class, you use the record keyword followed by the class name and a list of components (fields) enclosed in parentheses. The components define the state of the record.

public record Person(String name, int age) {
    // Body of the record class (optional)
}

Key Characteristics:

  1. Immutable: Record classes are immutable by default, meaning their state cannot be modified after creation. All fields are final and initialized through the constructor.
  1. Automatic Component Methods: Records automatically generate getter methods for each component field, allowing you to access the state of the record using dot notation (person.name()).
  1. Default equals(), hashCode(), and toString() Methods: Records automatically generate equals(), hashCode(), and toString() methods based on the components of the record, providing value-based semantics.
  1. Compact Syntax: Records provide a concise syntax for defining data classes, reducing boilerplate code and improving code readability.

Example:

public record Person(String name, int age) {}

// Usage
Person person = new Person("John", 30);
System.out.println(person.name());  // John
System.out.println(person.age());   // 30
System.out.println(person);         // Person[name=John, age=30]

Use Cases:

  • Data Transfer Objects (DTOs): Records are well-suited for representing DTOs in web services or data exchange formats like JSON.
  • Value Objects: Records can be used to model immutable value objects, such as coordinates, points, or currency amounts.
  • Domain Objects: Records are useful for representing domain entities with a fixed set of attributes and value-based semantics.

Considerations:

  • Records cannot extend other classes (except java.lang.Record) or have non-static fields or instance methods.
  • While records provide convenience for simple data classes, they may not be suitable for more complex classes requiring additional behavior or customization.

Compatibility:

  • Record classes were introduced as a preview feature in Java 14 and made stable in Java 16. They are fully supported in Java 17 and later versions.

In summary, record classes in Java provide a concise and convenient way to define simple data classes with immutable state, automatic component methods, and built-in equals(), hashCode(), and toString() implementations. They are particularly useful for modeling data-centric classes and reducing boilerplate code in Java applications.

// better alternative
// by default instance variables in a record class are private and final
// a record class cannot extend any other class but can implement any number of interfaces
// you can create a static instance variable inside a record class but cannot create a normal instance variable
// you can define your own methods in the record class
record Alien(int id, String name){
    public Alien{
        if(id==0) throw new IllegalArgumentException("id cannot be zero");
    }
}

public class Main {
    public static void main(String[] args) {
        Alien a1 = new Alien(1, "Probal Kar");
        Alien a2 = new Alien(1, "Probal Kar");
        Alien a3 = new Alien(2, "Awantika Ankushe");
        // Alien a4 = new Alien(0, "Smita Jagdale");    // throws error

        System.out.println(a1);
        System.out.println(a1.equals(a2));
        System.out.println(a1.equals(a3));

        System.out.println(a1.id());    // instance variables can be accessed by calling them as a method

    }
