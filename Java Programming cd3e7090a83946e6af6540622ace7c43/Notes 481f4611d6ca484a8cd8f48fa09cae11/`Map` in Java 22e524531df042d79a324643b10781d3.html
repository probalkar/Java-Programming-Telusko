In Java, a Map is an interface in the Java Collections Framework that represents a collection of key-value pairs, where each key is associated with exactly one value. The Java Collections Framework provides several implementations of the Map interface, each with its own characteristics and use cases. Additionally, Hashtable is a specific implementation of the Map interface that uses a hash table data structure to store key-value pairs. Here's an overview of the Map interface, along with a description of Hashtable:

Map Interface:

  1. Key-Value Pairs: A Map stores elements as key-value pairs, where each key is unique within the Map and is associated with exactly one value.
  1. No Duplicate Keys: Maps do not allow duplicate keys. If an attempt is made to insert a key that already exists in the Map, the new value will overwrite the existing value associated with that key.
  1. Fast Lookup: Maps provide fast lookup and retrieval of values based on their corresponding keys. This allows for efficient data retrieval and manipulation based on specific identifiers.

Common Methods of Map:

  • put(K key, V value): Associates the specified value with the specified key in the map.
  • get(Object key): Returns the value associated with the specified key in the map, or null if the key is not present.
  • containsKey(Object key): Returns true if the map contains a mapping for the specified key.
  • remove(Object key): Removes the mapping for the specified key from the map, if present.
  • keySet(): Returns a Set view of the keys contained in the map.

HashMap:

  • HashMap is one of the most commonly used implementations of the Map interface.
  • It uses a hash table to store key-value pairs, providing fast average-case performance for put, get, and remove operations.
  • HashMap does not guarantee the order of key-value pairs and allows null keys and null values.

TreeMap:

  • TreeMap is an implementation of the Map interface that maintains its entries in sorted order based on the natural ordering of its keys or a specified Comparator.
  • It uses a red-black tree data structure for storage, which provides log(n) time complexity for most operations.
  • TreeMap is suitable for scenarios where you need a sorted collection of key-value pairs.

LinkedHashMap:

  • LinkedHashMap is a HashMap subclass that maintains a doubly-linked list of entries, preserving the order of insertion.
  • It iterates over elements in the order in which they were inserted, providing predictable iteration order.
  • LinkedHashMap is useful when you need to maintain insertion order while still benefiting from the fast lookup of a hash table.

Hashtable:

  • Hashtable is a legacy implementation of the Map interface that is synchronized and thread-safe.
  • It uses a hash table data structure to store key-value pairs, providing similar functionality to HashMap.
  • Hashtable does not allow null keys or values and is less commonly used in modern Java applications due to its synchronization overhead and lack of flexibility compared to ConcurrentHashMap.

When to Use Maps:

  • Use Maps when you need to store and retrieve data based on key-value associations.
  • Maps are useful for representing relationships and associations between entities, such as dictionaries, caches, and configuration settings.
  • Choose the appropriate Map implementation based on your requirements for performance, ordering, synchronization, and null values.

Overall, Maps are a versatile and widely used data structure in Java, providing efficient storage and retrieval of key-value pairs. Choose the appropriate Map implementation based on your specific requirements for performance, ordering, and synchronization.

import java.util.HashMap;
import java.util.Hashtable;
import java.util.Map;

public class Main {
    public static void main(String[] args) {
        // Map<String, Integer> students = new HashMap<>();    // HashMap is not synchronized nor sorted
        Map<String, Integer> students = new Hashtable<>();  // Hashtable is synchronized but also not sorted
        students.put("Probal Kar", 49);
        students.put("Smita Jagdale", 56);
        students.put("Awantika Ankushe", 69);

        System.out.println(students);   // prints the map object in key value pairs
        System.out.println(students.keySet());  // prints set of keys in students object

        for(String s : students.keySet()){
            System.out.println(s + " : " + students.get(s));
        }
    }
}
